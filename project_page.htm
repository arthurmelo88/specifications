News

The core specification work on the openEHR specifications heading toward Release 1.0 has been done, with over 25 CRs having been processed by the ARB in Q4 2004 and Q1 2005. Two key additions remain: the redesigned Instruction and the openEHR Templates. With the addition of these, Release 1.0 will be declared sometime in May. The Instruction re-design is of key importance, and will provide a comprehensive model of the concepts INSTRUCTION, ACT_EXECUTION and INSTRUCTION_EXECUTION, supporting standardised querying (which Instructions are current in Active state?) and complex interventions, such as multi-drug therapy with suspend/abort conditions, as well as simple medication orders. Likewise, the Template specification will show how to create local templates based on archetypes, including local default values and other settings. Both of these documents will be published in draft form for community comment as soon as possible.

Heading towards Release 1.0 there are also some important CRs remaining. These can be viewed on the plone CR page. Once we have reached Release 1.0, the aim is to restrict changes to error corrections and new ITSs, ensuring stabiliy for implementors.  We have also significantly improved the organisation of documents, namespaces, and provided new or rewritten top-level documents to aid new-comers to openEHR, including those at an executive level.

 Since DIRECTORY is 1:1 with EHR, there is no need for a name or other LOCATABLE attrinbutes on DIRECTORY, since there can onlybe one per EHR. But for consistency in implementation of paths, it probably should inherit, and have a name fixed to either empty string or "directory".
