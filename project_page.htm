News

The specification work on Release 1.0 of the openEHR specifications is nearly complete. Two key additions remain: the redesigned Instruction and the openEHR Templates. AXML-schemas are in an advanced stage of development. Another 40 or so smaller CRs remain. With the addition of these, Release 1.0 will be declared at end 2005 or early 2006. The Instruction re-design is of key importance, and will provide a comprehensive model of the concepts INSTRUCTION, ACTION and INSTRUCTION_ACTION, supporting standardised querying (e.g. which Instructions are current in Active state?) and complex interventions, such as multi-drug therapy with suspend/abort conditions, as well as simple medication orders. Likewise, the Template specification will show how to create local templates based on archetypes, including local default values and other settings.

The CRs can be viewed on the plone CR page. Once we have reached Release 1.0, we will restrict changes to error corrections and new ITSs, ensuring stabiliy for implementors.  We have also significantly improved the organisation of documents, namespaces, and provided new or rewritten top-level documents to aid new-comers to openEHR, including those at an executive level.

How To...

Have a local up-to-date copy of the documents on your own machine

Just follow these steps using your subversion client:
  • create a directory where the documents will be saved, e.g. somewhere/openEHR/specifications
  • using TortoiseSVN or some other Subversion client, checkout  into this http://svn.openehr.org/specification/BRANCHES/Release-1.0-candidate/publishing/ directory. (See Subversion pages if you don't understand this).
  • the file index.html is top of the document tree, and will allow you to access all documents offline
  • updating to the latest is as easy as telling the subversion client to update that directory (this can be automated with a simple cron script)
