openEHR Specification Project
Release 1.0.1
Release Notes

Overview

Release 1.0.1 is the corrective release to Release 1.0. It fixes the typographical errors from Release 1.0, includes significant improvements in the explanatory text of the documents, and contains a small number of CRs. The list of CRs executed for this release can be seen in the Change History, and the distribution of CRs over specifications can be seen on the roadmap page. Most of the changes were done to improve the common.change_control model of versioning due to the Extract work that will appear in Release 1.1. The largest correction is to the date/time data types, which have now been aligned with the ISO 8601 standard. An EHR access control object has been added, along with an overall description of how security works. Details of privacy and access control are left to a future release. Other than that, the definition of various identifiers in support.identification was tightened up, and the use of identifiers made more precise.

Obtaining the Release Materials

You can access all deliverables in the release from the roadmap page, just by clicking on links. To get the entire release as a web tree, follow the instructions on how to use Subversion to do this.
