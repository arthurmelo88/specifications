openEHR Specification Project
Release 1.0.1 Release Notes

Overview

Release 1.0.1 is the corrective release to Release 1.0. It fixes the typographical errors from Release 1.0, includes significant improvements in the explanatory text of the documents, and includes around 50 CRs (see History). The majority of these have been raised by members of the openEHR community either due to detailed review, or due to working with implementations. The release now appears to be extremely stable. We believe this release is a true milestone in health computing.

Functional changes to the models include the following:
  • original documents can now be linked to or carried inline COMPOSITIONs, in a flexible way;
  • improvements in the rm.common.change_control model of versioning;
  • improved modelling in Quantity data types;
  • corrections to the date/time data types, which have now been aligned with the ISO 8601 standard;
  • an EHR access control object has been added, along with an overall description of how security works;
  • the definition of various identifiers in support.identification was tightened up, and the use of identifiers made more precise;
  • corrected inheritance errors in LOCATABLE;
  • improvements to resource_description package (rm.common.resource) for use in archetypes;
  • numerous improvements to openEHR Archetype profile, details of ADL, quoting and unicode.

Obtaining the Release Materials

You can access all deliverables in the release from the roadmap page, just by clicking on links. To get the entire release as a web tree, follow the instructions on how to use Subversion to do this.
