Release 1.0.1
Release Notes

Overview

Release 1.0.1 is a minor corrective release to Release 1.0. It fixes the typographical errors from Release 1.0, includes significant improvements in the explanatory text of the documents, and contains a small number of CRs. The list of CRs executed for this release can be seen in the Change History, and the distribution of CRs over specifications can be seen on the roadmap page. Most of the changes were done to improve the common.change_control model of versioning due to the Extract work that will appear in Release 1.1. Some minor improvements are made to the date/time data types. Other than that, the definition of various identifiers in support.identification was tightened up, and the use of identifiers made more precise.

Obtaining the Release Materials

You can access all deliverables in the release from the roadmap page, just by clicking on links. To get the entire release as a web tree, follow the instructions on how to use Subversion to do this.

Language 2.0
(ADL2)		 Draft ADL version 2.0 is an improved syntax for expressing archetypes, and is has two important
features over ADL 1.3: it is extensible without the syntax needing to be changed, because in ADL2,  an archetype is a dADL document; and due to being in dADL/embedded cADL  format, it can more easily be converted to and from XML.
The current version of this specification has not yet been fully implemented or tested. Implementors of this specification should consider engineering backwards compatibility for ADL 1.3 in their tools.
