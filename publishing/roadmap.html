openEHR Specification Project
Release 1.0.1 Documents

Overview

The following table provides access to the deliverables of the openEHR specification project. There are two groups of deliverables, as follows:
  • stable: the deliverable is considered complete at the current point in time, has been widely reviewed, and validated in implementation.
  • in-development: the deliverable is in active development, and implementers should only consider using it in a prototype or research mode.
Release notes for this release. The rules for change management between releases are given in the Release Strategy.

Reading the Documents

Stable Specifications

General

 Introducing openEHR First introduction to the openEHR Foundation and its activities.
CM Plan Technical document describing how versioning, changes, and releases are made. Describes the workflow of the Architectural Review Board (ARB).

Requirements

 ISO 18308 Conformance Statement Document describing conformance of openEHR architecture to ISO TS 18308, "Requirements for EHR Architectures".

Architecture

 Architecture Overview
(HTML Online version)		 “Read me 1st” document for the whole architecture. provides a summary of the reference, archetype and service models, and describes global semantics.
Modelling Guide Describes the use of UML and other related issues in object-oriented modelling.
Terminology Documentary form of the openEHR terminology, which is a set of vocabularies and code sets used by the reference and archetype models.

Reference Model

 EHR IM The information model of the openEHR EHR .
Demographic IM The openEHR demographic model.
Common IM Information model containing common concepts, including the archetype-enabling LOCATABLE class, party references, audits and attestations, change control, and authored resources.
Data Structures IM Information model of data structures, incuding a powerful model of HISTORY and EVENT.
Data Types IM Information model of data types, including quantities, date/times, plain and coded text, time specification, multimedia and URIs.
Support IM Support model containing low-level concepts, assumed types, and terminology interface specification used in the rest of the models.
Integration IM Model supporting expression of legacy data in a free form for further processing into and out of openEHR information structures.

Archetype Object Model (AOM) Object model of archetypes.
openEHR Archetype Profile (OAP) openEHR plug-in additions to the generic archetype object model.
