openEHR Specifications

The openEHR Specifications project generates the requirements, normative specifications and implementation technology specifications of the openEHR computing architecture for health records. It also develops a number of guides. The deliverables are summarised as follows.

Guides

There are three important guiding documents in the specification project:

  • Introduction to openEHR: an organisational overview
  • The openEHR Roadmap: technical overview of the specifications
  • The openEHR Modelling Guide: the formalisms and modelling methods used in openEHR.

Requirements

The requirements part of the project develops requirements for the EHR, and indicates correspondence of the openEHR specifications to those requirements as well as published requirements material.

Architecture

The architecture part of the project develops abstract, formal models of the knowledge-enabled EHR.

Implementation Technology Specifications

The ITS part of the project converts the architecture models to concrete expressions in implementation technologies, such as XML formalisms, programming languages, distributed object formalisms and database schemas.
