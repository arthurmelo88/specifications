openEHR Specifications - Release 1.0 Candidate

This version of the specifications is under development as the candidate for Release 1.0 of the openEHR Specification Project. Not all content changes have yet been approved by the openEHR ARB.

The openEHR Specifications project generates the requirements, normative specifications and implementation technology specifications of the openEHR computing architecture for health records. It also develops a number of guides. All deliverables available in the explorer tree to the left. Deliverables containing changes with respect to current latest release are listed below.

Architecture

The architecture part of the project develops abstract, formal models of the knowledge-enabled EHR.

New drafts:

  • EHR Information Model
  • Data Structures Information Model
  • Archetype Definition Language (ADL)
  • Archetype Object Model (AOM)

Implementation Technology Specifications

The ITS part of the project converts the architecture models to concrete expressions in implementation technologies, such as XML formalisms, programming languages, distributed object formalisms and database schemas.

New drafts:

  • XML-schemas: 
    • BaseTypes - Support, Data Types, Common; document form; XSD file
    • Structures - LOCATABLE + Data Structures; document form; XSD file
    • Content - descendants of CONTENT_ITEM; document form; XSD file
    • Composition - COMPOSITION class and friends; document form; XSD file
