openEHR Specification Repository Publishing Area

Requirements project

 The openEHR Requirements project. This project develops:
  • requirements for EHRs
  • conformance statements to requirements standards, e.g. ISO TS 18308

Architecture project

 The openEHR Architecture project. This project develops the main specifications for the openEHR EHR, including
  • Information Model, Archetype Model, Sevice models for the EHR, EHR Extract, Demographics
  • Information Model and Archetype Models for Data types, Data Structures
  • Archetype Definition Language

Engineering project

 The openEHR Engineering project. This project develops the Implementation Technology Specifications (ITSs) - computable expressions of the abstract specifications of the Architecture project. These include:
  • XML-schemas
  • IDL expressions
  • programming language expressions
  • database schema language expressions

Configuration management

This is the CM area of the openEHR specification repository. It contains:
  • the openEHR Change Management Plan (CMP)
  • Change Requests (CRs)
  • Problem Reports (PRs)
